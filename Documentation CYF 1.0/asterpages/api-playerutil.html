The PlayerUtil object [E/M/W]



This is for modifying all UI that displays the Player's status at the same time. You can also modify each UIs by using the variables.


* For v0.5.3 Update, Asterisk Mod has changed Player's UI drastically.
* As a consequence of that, the most variables and functions of PlayerUtil is very changed and those name
   is also changed.
* It's too changed, so I divided the pages on old version (v0.5.2.x = before v0.5.3) and new version.
* PlayerUtil of old version is here.


< Technical info >
In AsteriskMod v0.5.3, It does NOT use original CYF's object to display the Player's status.
CYF uses text as TextManager class, but AsteriskMod adds new class UIStaticTextManager class and uses to display the status. (See also The StaticText Object.)
The Player's LifeBar uses other added class PlayerLifeBar. (See also The LifeBar Object there is below this page.)

The GameObject "*Name" and "*Love" is used instead of The GameObject "NameLv"; that is the parent of, the letters of the Player's Name and Love.
The GameObject "*HPRect" is used instead of The GameObject "HPRect" that conrols displaying of the Player's HP.

"UIStats" class that controls Displaying of the player's status has divided to "PlayerNameText", "PlayerLoveText" and "PlayerLifeUI".
There are each classes relating the Player's UI in Assets/Scripts/AsteriskMod/UIStats folder.

namespace of PlayerUtil is changed AsteriskMod.Lua to AsteriskMod.
(However, namespace will be not changed for compatibility if the Mod will be launched as before v0.5.3.)

Note that this feature can NOT be used in Overworld because of some reasons.
That's why original UIStats class and that objects is not deleted completely.


Status UI (All)




PlayerUtil.x Horizontal position of the all UI's parent object, relative to the initial position.

This is NOT same to call .Move() each UIs.

This function moves only parent object of all the player's UIs.
In other words, moves reference positions of the UIs.


PlayerUtil.y Vertical position of the all UI's parent object, relative to the initial positon.

This is NOT same to call .Move() each UIs.

This function moves only parent object of all the player's UIs.
In other words, moves reference positions of the UIs.


PlayerUtil.Move( x, y) Moves the all UI showing the player's status x pixels right, and y pixels up.

See also PlayerUtil.x and PlayerUtil.y.


PlayerUtil.MoveTo( x, y) Moves the all UI showing the player's status to a new position, relative to the initial position.

(Same as setting PlayerUtil.x and PlayerUtil.y simultaneously.)
See also PlayerUtil.x and PlayerUtil.y.


PlayerUtil.MoveToAbs( absx, absy) Moves the all UI showing the player's status to a new position, relative to the bottom left corner of the window.


PlayerUtil.SetControlOverride( active) Either true or false. If true, bars and texts showing the Player's status do NOT update anymore.
You should call this if you modify objects displaying the Player's status.
This is same to call PlayerUtil.Name.SetControlOverride() PlayerUtil.Love.SetControlOverride() and PlayerUtil.SetHPControlOverride() at the same time.


Status UI (Name & Love)




StaticText PlayerUtil.Name (Read-only)
 orStaticText PlayerUtil.nametext (Read-only) Returns propety of the text showing the Player's name, as a StaticText object.
You can not use PlayerUtil.Name.Remove() and PlayerUtil.Name.DestroyText() in this object.


PlayerUtil.SetJapaneseNameActive( active) Supports Japanese player name.
日本語訳や詳細は Japanese Localization [日本語対応]を参照してください。


StaticText PlayerUtil.Love (Read-only)
 orStaticText PlayerUtil.lovetext (Read-only)
 Returns propety of the text showing the Player's love, as a StaticText object.
The position of this object is changed with the length of name's text automatically until PlayerUtil.Love.SetControlOverride(true) is called.
You can not use PlayerUtil.Love.Remove() and PlayerUtil.Love.DestroyText() in this object.


PlayerUtil.SetLV( lv) Set the player's LV as string.
This means you can set the LV other than numbers.
This is same to call PlayerUtil.Love.SetText().

Note that you can not use Text commands.


Status UI (HP)



On AsteriskMod, positions of the UIs showing the HP is fixed. (Because of More UNDERTALE-ish)
(This means, the UIs do NOT is moved eben if (the length of) the texts showing name and love is changed.)


PlayerUtil.hpx
or PlayerUtil.HPx Parent object's horizontal position of the objects showing the player's HP, relative to the initial position.

This is NOT same to call .Move() each UIs.

This function moves only parent object of HP's UIs.
In other words, moves reference positions of the UIs.


PlayerUtil.hpy
or PlayerUtil.HPy Parent object's vertical position of the objects showing the player's HP, relative to the initial position.

This is NOT same to call .Move() each UIs.

This function moves only parent object of HP's UIs.
In other words, moves reference positions of the UIs.


PlayerUtil.HPMove( x, y) Moves the all UI showing the player's HP x pixels right, and y pixels up.

See also PlayerUtil.hpx and PlayerUtil.hpy.


PlayerUtil.HPMoveTo( x, y) Moves the all UI showing the player's HP to a new position, relative to the initial position.

(Same as setting PlayerUtil.hpx and PlayerUtil.hpy simultaneously.)
See also PlayerUtil.hpx and PlayerUtil.hpy.


sprite PlayerUtil.HPLabel (Read-only)
 orsprite PlayerUtil.hplabel (Read-only)
 Returns propety of the objects written "HP", as a sprite object.
You can not use PlayerUtil.HPLabel.Remove() in this object.


LifeBar PlayerUtil.HPBar (Read-only)
 orLifeBar PlayerUtil.hpbar (Read-only)
 Returns propety of the bar objects showing the Player's HP, as a LifeBar object.
You can not use PlayerUtil.HPBar.Remove(), PlayerUtil.HPBar.layer, PlayerUtil.HPBar.MoveBelow(), PlayerUtil.HPBar.MoveAbove(), PlayerUtil.HPBar.SendToBottom() and PlayerUtil.HPBar.SendToTop() in this object.


LifeBar PlayerUtil.CreateLifeBar( layer = "BelowHPBar") Creates Life Bar and Return The LifeBar Object.
On The LifeBar Object, you can specify special layer named "BelowHPBar" and "AboveHPBar"


StaticText PlayerUtil.HPText (Read-only)
 orStaticText PlayerUtil.hptext (Read-only)
 Returns propety of the text showing the Player's HP, as a StaticText object.
The position of this object is changed with the length of player's life bar automatically until PlayerUtil.HPText.SetControlOverride(true) is called.
You can not use PlayerUtil.HPText.Remove() and PlayerUtil.HPText.DestroyText() in this object.


PlayerUtil.SetHP( hp, maxhp, updateText = true) Sets the lengthes of bars displaying the Player's HP from 2 values.

  • updateHPText: If true, sets the text showing the Player's HP as based on 2 values.
This is similar to call PlayerUtil.HPBar.SetHP() and PlayerUtil.SetHPText() at the same time.
Recommends to call PlayerUtil.SetHPControlOverride(true) or call PlayerUtil.HPBar.SetControlOverride(true) and PlayerUtil.HPText.SetControlOverride(true) before use this.


PlayerUtil.SetHPText( hp, maxhp) Sets the text showing the Player's HP as based on 2 values.
Recommends to call PlayerUtil.SetHPControlOverride(true) or PlayerUtil.HPText.SetControlOverride(true) before use this.
You should use PlayerUtil.HPText.SetText() if you set the text as string.


PlayerUtil.SetHPControlOverride( active) Either true or false. If true, bars and texts showing HP do NOT update anymore.
You should call this if you modify objects displaying the Player's HP.
This is same to PlayerUtil.HPBar.SetControlOverride() and PlayerUtil.HPText.SetControlOverride() at the same time.

Default by 3.


PlayerUtil.defaultinvtime Gets ot Sets default invincible time of the player.
Default by 1.7.


PlayerUtil.GetSoulAlpha() Gets the alpha (transparency) of the player's soul.
When the player damaged, the soul will be blinking. This function can be get that blinking as number 0 or 1.

FIGHT UI



PlayerUtil.SetTargetSprite( path) Change a sprite's current image of target in FightUI.


PlayerUtil.SetTargetChoiceSprite( path) Change a sprite's current image of targetchoice in FightUI.


PlayerUtil.SetTargetChoiceAnim( anim, frequency, prefix = "", forceSetAnimation = true) Sets the animation of targetchoice's sprite. Use it like sprite.SetAnimation().

if forceSetAnimation is true, swaps animation if the animation of targetchoice is running.


PlayerUtil.ResetTargetChoiceAnim( forceSetAnimation = true) Resets the animation of targetchoice's sprite to the default slashing animation.

If forceSetAnimation is true, swaps animation if the animation of targetchoice is running.


PlayerUtil.GetTargetChoiceX() Get horizontal position of targetchoice.
This is useful for PlayerUtil.SimulateDamage().

If targetchoice is not shown or not stopped (current state is not ATTCKING or the player does not attack by pressing Z(Enter) Key), returns the edge of targetchoice's position.


PlayerUtil.SimulateDamage( playerATK, playerWeaponATK, enemyDEF, targetChoiceX = 0, randomValue = -1) Simulate an enemy's damage with the arguments.

  • playerATK: player's attack, as integer value.
  • playerWeaponATK: attack of player's weapon, as integer value.
  • enemyDEF: enemy's defence, as integer value.
  • targetChoiceX: horizontal position of targetchoice, as float value.
    recommended to set between-273.5(left end) and 273.5(right end)
    0 means center.
  • randomValue: random additional attack, as float value between0.0 and 1.0.
    If sets out of range, it is set random value.


The LifeBar Object



LifeBars are composed of 3 GameObjects. Those are parent, BGBar, and FillBar.
You don't need to think about the parent object.
BGBar(Background Bar) means the bar placed far side, as its name suggests, default color is red.
On the other hand, FillBar means the bar placed nearer side, default color is yellow.
Simply put, BGBar is the bar showing the Player's MaxHP and FillBar is the bar showing current the Player's HP.
This is not only used in PlayerUtil.HPBar, but also you can create LifeBar by calling PlayerUtil.CreateLifeBar() and use that.



lifeBar.controlOverride If true, the HPBar displaying the player's HP does not update anymore.
This is for PlayerUtil.HPBar.
This parameter of lifebars created by PlayerUtil.CreateLifeBar() is always true.


lifeBar.SetControlOverride( active) Same as setting lifeBar.controlOverride.

See lifeBar.controlOverride.


lifeBar.isactive Tells you if a LifeBar object is active.


lifeBar.Remove() This function destroys a LifeBar object, similar to .Remove() for bullets and sprites. Trying to get, set or call almost anything besides LifeBar.isactive on a destroyed LifeBar object will error.


lifeBar.limited Default by true.
LifeBars are limited a length of it.
Normally, FillBar's length is not over to BGBar's one, and Both Bars' lengthes are fixed if value is over 100.
If this parameter is false, those limit is disabled.


lifeBar.hp Gets or sets a length of FillBar.
This is not percentage.
If lifeBar.limited is true, FillBar's length is not over BGBar's one.


lifeBar.maxhp Gets or sets a length of BGBar.
This means getting or setting maximum value of the bars' lengthes.
If lifeBar.limited is true and this value is over 100, BGBar's length is fixed.


lifeBar.SetHP( newHP, newMaxHP) Same as setting lifeBar.hp and lifeBar.maxhp simultaneously.

See lifeBar.hp and lifeBar.maxhp.


lifeBar.height Gets or sets a height of the bar.
Default by 20.


, , , = 1 lifeBar.bgcolor Gets or sets the coloration of a BGBar, as a table of 3 or 4 values from 0 to 1.

For example, lifeBar.bgcolor = {1.0, 0.0, 0.0} colors the BGBar red.

You can provide a 4th value, which sets the alpha (transparency) of the BGBar.


, , , = 255 lifeBar.bgcolor32 Gets or sets the coloration of a BGBar, as a table of 3 or 4 values from 0 to 255.

For example, lifeBar.bgcolor32 = {255, 0, 0} colors the BGBar red.

You can provide a 4th value, which sets the alpha (transparency) of the BGBar.


lifeBar.bgalpha Gets or sets a BGBar's transparency, as a value from 0 to 1.


lifeBar.bgalpha32 Gets or sets a BGBar's transparency, as a value from 0 to 255.


, , , = 1 lifeBar.fillcolor Gets or sets the coloration of a FillBar, as a table of 3 or 4 values from 0 to 1.

For example, lifeBar.fillcolor = {1.0, 1.0, 0.0} colors the FillBar yellow.

You can provide a 4th value, which sets the alpha (transparency) of the FillBar.


, , , = 255 lifeBar.fillcolor32 Gets or sets the coloration of a FillBar, as a table of 3 or 4 values from 0 to 255.

For example, lifeBar.fillcolor32 = {255, 255, 0} colors the FillBar yellow.

You can provide a 4th value, which sets the alpha (transparency) of the FillBar.


lifeBar.fillalpha Gets or sets a FillBar's transparency, as a value from 0 to 1.


lifeBar.fillalpha32 Gets or sets a FillBar's transparency, as a value from 0 to 255.


lifeBar.x Horizontal position of lifeBar relative to the initial position.


lifeBar.y Vertical position of lifeBar relative to the initial position.


lifeBar.Move( x, y) Moves the lifeBar x pixels right, and y pixels up.

See lifeBar.x and lifeBar.y.


lifeBar.MoveTo( x, y) Same as setting x and y simultaneously.

See lifeBar.x and lifeBar.y.


lifeBar.layer Gets or sets the current layer a bar is on. Does nothing(?) if you set it to a layer that doesn't exist.

Default value: "BelowHPBar"

You can specify special layer named "BelowHPBar" and "AboveHPBar".
On PlayerUtil.HPBar, layer can't be changed.


lifeBar.SendToTop() Sends this bar to the top of its bar's hierarchy. This is same to sprite.SendToTop().
PlayerUtil.HPBar.SendToTop() can't be used.


lifeBar.SendToBottom() Sends this bar to the bottom of its bar's hierarchy. Similar rules apply as with SendToTop().
PlayerUtil.HPBar.SendToBottom() can't be used.


lifeBar.MoveBelow(lifeBar otherLifeBar) Moves a life bar object below another life bar object.
They must both be on the same layer.
PlayerUtil.HPBar.MoveBelow() can't be used.


lifeBar.MoveAbove(lifeBar otherLifeBar) Moves a life bar object above another life bar object.
They must both be on the same layer.
PlayerUtil.HPBar.MoveAbove() can't be used.

Additional Setup



Currently, Asterisk Mod adds scripts are set up like this:

  • State scripts - will contain some functions. You can set the behavior of the state you create. (See CustomState section for more information.)
  • GlobalsScripts scripts - will contain only [E/M/W]([E/M/W/S]) functions. (See GlobalScript section for more information.)

Mod's Information



This setup is important for the compatibility with prev AsteriskMod's version.
Furthermore, you can add a description and change mod's title by using this.
This is not required feature, but I recommend to setup this.

Since v0.5.3.1, info.cyfmod is required to the mod made with AsteriskMod because the engine will get to distinguish CYF's mods and AsteriskMod's mods.


Basically what you should do is make a file and write parameters in it.

At first, create a file named "info.cyfmod" in your mod's root folder.
For example:

-- directory:
YOURMOD
├─ Audio
├─ Lua
├─ Sounds
├─ Sprites
└─ info.cyfmod -- this (is <= 1MB)

.cyfmod file's format is similar to .ini file's that.
This format supports only string and table(array) of string.

There is .cyfmod file's format: 
;;this is comment out.
;;string
key="parameter"
;;table (can use since v0.5.3)
key={"element1", "element2"}
;;You can use some escape sequence since v0.5.3.
NewLine="\n"
CarriageReturn="\r"
;;You can use section in this format, but info.cyfmod ignores section.

There are available keys in info.cyfmod:

  • target-version - sets AsteriskMod's version that your mod supports.
    If you have set this parameter correctly, the engine works as old version.
    There are supported versions below:
    • v0.5.3
    • v0.5.2.9
    • v0.5.2.8
  • encounter-names - replaces encounters' names.
    Array's indexes correspond to Lua files' order.
    Name will not be override by specifying empty string.
  • show-encounters or show - limits encounters the player can select.
    You can choose encounters you wanna SHOW by using this.
    Each element is the file's name (without extension) of encounter.
    show-encounters parameter has a priority to hide-encounters parameter
  • hide-encounters or hide - limits encounters the player can select.
    You can choose encounters you wanna HIDE by using this.
    Each element is the file's name (without extension) of encounter.
    show-encounters parameter has a priority to hide-encounters parameter
  • supported-languages - sets languages mod supports, displaying on mod selection.
    The Engine automatically displays supported languages by detecting .lang file in mod's Localization folder.
    This parameter is override of that displaying.
     You can check available languages in Language variable (in Special Variables section).
  • title - replaces the mod's title.

  • subtitle - replaces the text that is below mod's title text.

  • description - sets your mod's description.

  • description-file - sets the path of the file written the description.
    description-file parameter has a priority to description parameter

  • description-align - sets alignment of the description. UpperLeft by default.
    You can choose the paramter in the following:
    • UpperLeft
    • UpperCenter
    • UpperRight
    • MiddleLeft
    • MiddleCenter
    • MiddleRight
    • LowerLeft
    • LowerCenter
    • LowerRight
  • font - sets the font of mod's title and sub-title(the number of encount).
    You can choose the paramter in the following:
    • 8bitoperator
    • pixeloperator
  • bg-color - sets the background's color, as a table of 3 or 4 values from 0 to 1.
  • bg-color32 - sets the background's color, as a table of 3 or 4 values from 0 to 255.

  • bg-alpha - sets the background's alpha (transparency), as value from 0 to 1.

  • bg-alpha32 - sets the background's alpha (transparency), as value from 0 to 255.
  • launch-bg-color - sets the color of the background when the mod is launching, as a table of 3 or 4 values from 0 to 1.
  • launch-bg-color32 - sets the color of the background when the mod is launching, as a table of 3 or 4 values from 0 to 255.
  • launch-bg-alpha - sets the alpha (transparency) of the background when the mod is launching, as value from 0 to 1.
  • launch-bg-alpha32 - sets the alpha (transparency) of the background when the mod is launching, as value from 0 to 255.

Invalid format is ignored on loading file.

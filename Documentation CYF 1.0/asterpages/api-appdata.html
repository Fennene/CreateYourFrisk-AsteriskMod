The AppData object [E/M/W/S]


namespace: AsteriskMod
code: AsteriskMod/File/AppDataFileManager.cs


The AppData Object provides IO of AppData.
You can use a directory of CYF's AppData.
On Windows, the path of that directory is C:\Users\%USERNAME%\AppData\LocalLow\Nobody's Getting Paid Here\Create Your Frisk - Asterisk Mod\Mods\[YOUR_MOD_DIR_NAME].



AppData.OpenFile( path, mode = "rw") returns File This function opens a file in your Mod's AppData folder ONLY, for reading and writing.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.

  • mode: Determines how the file should be opened:
    • r: The file can only be read from.

    • w: The file can only be written to.

    • rw: The file can be read from AND written to.

    Files can be created through use of File.Write.

See The Misc Object for more info about The file object.


AppData.FileExists( path) Use this function to check if a file in your Mod's AppData folder ONLY exists.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.

Use this before AppData.OpenAppDataFile if you plan to read from a file.


AppData.DirExists( path) Use this function to check if a folder in your Mod's AppData folder ONLY exists.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.


AppData.ListDir( path, getFolders = false) This function returns a list of either files or folders in your Mod's AppData folder ONLY.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.

  • getFolders: If true, this function will return the names of all Folders in path.
    Otherwise, this function will return the names of all Files in path.


AppData.CreateDir( path) Attempts to create a folder with the path path relative to your Mod's AppData folder.
Using ../ to go up a folder is NOT allowed.

Returns true if the operation was successful, false otherwise.


AppData.MoveDir( path, newPath) Attempts to move a folder with the path path relative to your Mod's AppData folder to a new location newPath also relative to your mod's AppData folder.
Using ../ to go up a folder is NOT allowed.

Returns true if the operation was successful, false otherwise.


AppData.RemoveDir( path, force = false) Attempts to delete a folder with the path path relative to your Mod's AppData folder.
Using ../ to go up a folder is NOT allowed.

  • force: If true, this function will forcefully remove a folder. Otherwise, a folder will only be removed if it is empty.
Returns true if the operation was successful, false otherwise.


AppData.RemoveAll() Attempts to delete your Mod's AppData folder.
Returns true if the operation was successful, false otherwise.


iniFile AppData.OpenIniFile( path)
 or iniFile AppData.OpenFileAsIni( path) This function opens a file in your Mod's AppData folder ONLY, as ini file.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.

You can open file even if file does not exist.
This function returns a iniFile object representing the file:


The iniFile object [E/M/W/S]


  • iniFile.Load(): loads a file to temporary ini data.
    You can call this even if file does not exist.
    This is called automatically when AppData.OpenIniFile() and AppData.OpenFileAsIni() is called.

  • iniFile.Save(): save temporary ini data to a file.

  • iniFile.FileExists(): returns whether a file exists or not.

  • iniFile.SectionExists( sectionName): returns whether the section exists or not.

  • iniFile.ParameterExists( sectionName, parameterName): returns whether the section and parameter exist or not.

  • iniFile.GetString( sectionName, parameterName): gets the parameter from sectionName and parameterName.
    Returns "" (empty string) if section or parameter do not exist.

  • iniFile.GetNumber( sectionName, parameterName): gets the parameter as number from sectionName and parameterName.
    Returns 0 if section or parameter do not exist.

  • iniFile.SetString( sectionName, parameterName, parameter): sets the parameter to temporary ini data.
    You should call iniFile.Save() if you wanna save to file.
    If section or parameter do not exists, add that automatically.

  • iniFile.SetNumber( sectionName, parameterName, parameter): sets the parameter to temporary ini data.
    You should call iniFile.Save() if you wanna save to file.
    If section or parameter do not exists, add that automatically.

  • iniFile.RemoveSection( sectionName): removes the section.
    If section does not exist, nothing happened.

  • iniFile.RemoveParameter( sectionName, parameterName): removes the parameter.
    If section or parameter do not exist, nothing happened.

  • iniFile.ChangePath( path, reload = false): changes the path of file.

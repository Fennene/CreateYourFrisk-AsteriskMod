Available Game Events in CustomState



There are below functions run in at some time point, you can write the script's code concisely with that.



StateStarting( oldState) Happens once when entered CustomState. You should do createing sprites/projectiles/etc, displaying text in dialog box or changing the player's visible if you want to that.

This function is optional.
This function is required until AsteriskMod v0.5.2.9.


StateEnding( newState) Happens once when entered other state. You should remove sprites/projectiles if there are create sprites/projectiles/Arena/etc in StateStarting().

This function is optional.
This function is required until AsteriskMod v0.5.2.9.


Update() Optional.
This function runs for every frame (usually at 60FPS, depends on the player's framerate) during be CustomState.


HandleAction() Optional.
This function runs when the player press Z/Enter key.


HandleArrows( left, right, up, down) Optional.
This function runs when the player press arrows key.

  • left: whether the player pressed left arrow key or not.

  • right: whether the player pressed right arrow key or not.

  • up: whether the player pressed up arrow key or not.

  • down: whether the player pressed down arrow key or not.


HandleCancel() Optional.
This function runs when the player press X/Shift key.


HandleCancel() Optional.
This function runs when the player press X/Shift key.


OnHit(bullet bullet) Every time a bullet collides with a player, this function gets called from the script that created the projectile. The bullet object in this function can be modified if you feel like it. For more information on the bullet object, see the section Projectile Management.

If you implement this function in your script, you have to manually define what should happen after bullet collision. This is what allows you to create orange, cyan and green projectiles, and much much more. If you don't implement this function in your script, it'll stick to the default of dealing 3 damage on hit.
the default damage of the player can be changed with PlayerUtil.defaultdamage.

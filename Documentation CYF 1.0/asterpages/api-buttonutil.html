The ButtonUtil object [E/M/W]



This is for accessing to each buttons or for changing parameters of buttons at the same time.

< Technical info >
The codes relating the buttons are moved to other new file and cleaned up in v0.5.3.
namespace is changed AsteriskMod.Lua to AsteriskMod.
(However, namespace will be not changed for compatibility if the Mod will be launched as before v0.5.3.)
UIController.cs have not contained functions of controlling each buttons. Instead of that, AsteriskMod.ButtonManager.cs controls all button and AsteriskMod.ActionButton.cs controls each buttons finely.



Button ButtonUtil.FIGHT (Read-only) Returns property of FIGHT Button as Button object.


Button ButtonUtil.ACT (Read-only) Returns property of ACT Button as Button object.


Button ButtonUtil.ITEM (Read-only) Returns property of ITEM Button as Button object.


Button ButtonUtil.MERCY (Read-only) Returns property of MERCY Button as Button object.


Button ButtonUtil[ buttonID] (Read-only) Returns Button object from specified button.
buttonID should be between 1(FIGHT) and 4(MERCY).


ButtonUtil.SetActives( fight = true, act = true, item = true, mercy = true) Sets the activates of all buttons at once.

  • fight: active of FIGHT Button.
  • act: active of ACT Button.
  • item: active of ITEM Button.
  • mercy: active of MERCY Button.

* Note: you can NOT inactive all buttons.


ButtonUtil.SetSprites( buttonSpritePathRoot, autoResize = false) Sets the sprites of all buttons at once by specifying directory.
buttonSpritePathRoot need to contain below named png file:

Button Name Sprite of Inactive Sprite of Active
FIGHT fightbt_0 fightbt_1
ACT actbt_0 actbt_1
ITEM itembt_0 itembt_1
MERCY mercybt_0 mercybt_1
autoResize: Whether button's size will be changed based on sprite's size or not.

autoResize is always true.


ButtonUtil.SetColors( r, g, b, a = 1) Sets colors of all button at the same time.
Sets the button's color, as 3 or 4 values from 0 to 1.
For example, ButtonUtil.SetColors(1.0, 0.0, 0.0) makes the button red.
Black areas are not affected by coloration.
The 4th value is the alpha (transparency) value of the button.


ButtonUtil.SetColor32s( r, g, b, a = 255) Sets colors of all button at the same time.
Sets the button's color, as 3 or 4 integer values from 0 to 255.
For example, ButtonUtil.SetColor32s(255, 0, 0) makes the button red.
Black areas are not affected by coloration.
The 4th value is the alpha (transparency) value of the button.


ButtonUtil.RevertAll( revertPosition = true) Revert the parameters of all button.
If revertPosition is false, positions of buttons will not be reverted.

revertPosition is always true.


The Button object



This is for controlling each buttons for example moves, recolors, and changes sprites.



button.x Horizontal position of sprite relative to the initial position.

Note that each buttons have different anchor of the position.
The anchor of FIGHT and ACT is bottom left anchor.
The anchor of ITEM and MERCY is bottom right anchor.


button.y Vertical position of sprite relative to the initial positon.

Note that each buttons have different anchor of the position.
The anchor of FIGHT and ACT is bottom left anchor.
The anchor of ITEM and MERCY is bottom right anchor.


button.absx Horizontal position of sprite relative to the bottom left corner of the screen.

The anchor of a button's absolute position is always the center of the texture as like sprite object.


button.absy Vertical position of sprite relative to the bottom left corner of the screen.

The anchor of a button's absolute position is always the center of the texture as like sprite object.


button.xscale Horizontal scaling of sprite (1.0 by default). 2.0 is twice as large, 0.5 is half as large.


button.yscale Vertical scaling of sprite (1.0 by default). 2.0 is twice as large, 0.5 is half as large.


button.isactive (read-only) Returns whether the button is active or inactive.

checks the active by using button.GetActive() instead of this.


, , , = 1 button.color Gets or sets the coloration of a button, as a table of 3 or 4 values from 0 to 1.

For example, button.color = {1.0, 0.0, 0.0} colors the button red. This actually overlays the button's original color, so if you want full control over the color, make sure your button is white. Black areas are not affected by coloration.

You can provide a 4th value, which sets the alpha (transparency) of the button.

changes color by using button.SetColor() instead of this. There is no method of getting color.


, , , = 255 button.color32 Gets or sets the coloration of a button, as a table of 3 or 4 integer values from 0 to 255.

For example, button.color32 = {255, 0, 0} colors the button red. This actually overlays the button's original color, so if you want full control over the color, make sure your button is white. Black areas are not affected by coloration.

You can provide a 4th value, which sets the alpha (transparency) of the button.

changes color by using button.SetColor32() instead of this. There is no method of getting color.


button.alpha Gets or sets a button's transparency, as a value from 0 to 1.


button.alpha32 Gets or sets a button's transparency, as a integer value from 0 to 255.


button.SetActive( active) Either true or false. If false, a button disappear and the player can not select it.
* Note: you can NOT inactive all buttons.

You should use button.alpha just if you just wanna hide a button.


button.SetSprite( normalSpritePath, overrideSpritePath, prefix = "UI/Buttons", autoResize = false) Change a button's current image. It retains its scaling.

  • normalSpritePath: the image while the player doesn't select.
  • overrideSpritePath: the image when the player select.
  • prefix: adds at the beginning of the above two string.
    prefix is assgined "UI/Buttons" if you do not assgin.
    prefix is assgined empty string if you do not assgin.
  • autoResize: Whether the size of a button's object will be resized based on the size of a image or not.
    autoResize is always true.


button.Move( x, y) Moves the sprite x pixels right, and y pixels up.

See button.x and button.y.


button.MoveTo( x, y) Same as setting x and y simultaneously.

See button.x and button.y.


button.MoveToAbs( x, y) Same as setting absx and absy simultaneously. Moves a sprite to an absolute screen position.

See button.absx and button.absy.


button.Scale( xScale, yScale) Same as setting xscale and yscale simultaneously.

See button.xscale and button.yscale.

This function is .


button.Revert( revertPosition = true) Revert the all parameter of a button.
If revertPosition is false, positions of buttons will not be reverted.

revertPosition is always true.



button.ResetColor() Resets the button's color.
This is same as button.SetColor(1, 1, 1, 1) or button.SetColor32(255, 255, 255, 255).


button.SetSize( width, height) Sets the size of the button's texture.

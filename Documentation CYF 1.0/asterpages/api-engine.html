The Engine object [E/M/W/S/GS]


namespace: AsteriskMod
code: AsteriskMod/CYFEngine.cs


The Engine Object provides functions to control CYF engine-self.



Engine.SetTargetFrameRate( frameRate = 60) Sets the frame rate of CYF. The frame rate should be integer number as between 30 and 120.
You can reset the frame rate by calling this without any arguments.

You can assign any positive number.



Engine.ModExists( modFolderName) Checks to exists that named mod.
Using ../ to go up a folder is NOT allowed.


Engine.RemoveAllProjectiles()
 or Engine.RemoveAllBullets() Attempts to remove all projectiles (bullet objects).


Engine.SetSpriteFilterMode( filename, filtermode = "POINT") Sets sprites' filter mode.
filtermode can be only POINT, BILINEAR, or TRILINEAR.
Default by "POINT".


Engine.SetProjectilesAutoRemovingActive( active) Default by true.
If false, the engine will not remove projectiles when wave is ended.


Engine.SetBulletPoolSize( newSize = 100) Sets the size of bullet pool. The size should be positive integer number
bullet pool is one of CYF's system for the bullet object. The number of creating the bullet object is limited, the number of that is 100.
You can call this for more bullet.

You can reset the size of bullet pool by calling this without any arguments.

Since v0.5.3.1(v0.5.3.0.2), IO of a mod's AppData is migrated to The AppData Object.


Engine.OpenAppDataFile( path, mode = "rw") returns File This function opens a file in your Mod's AppData folder ONLY, for reading and writing.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    (On Windows, This relative path is C:\Users\%USERNAME%\AppData\LocalLow\Nobody's Getting Paid Here\Create Your Frisk - Asterisk Mod\Mods\YOURMOD)
    Can enter subfolders.

  • mode: Determines how the file should be opened:
    • r: The file can only be read from.

    • w: The file can only be written to.

    • rw: The file can be read from AND written to.

    Files can be created through use of File.Write.

See The Misc Object for more info about The file object.


Engine.AppDataFileExists( path) Use this function to check if a file in your Mod's AppData folder ONLY exists.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.

Use this before Engine.OpenAppDataFile if you plan to read from a file.


Engine.AppDataListDir( path, getFolders = false) This function returns a list of either files or folders in your Mod's AppData folder ONLY.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.

  • getFolders: If true, this function will return the names of all Folders in path.
    Otherwise, this function will return the names of all Files in path.


Engine.AppDataDirExists( path) Use this function to check if a folder in your Mod's AppData folder ONLY exists.
Using ../ to go up a folder is NOT allowed.

  • path: Path to the file to open, relative to your Mod's AppData folder.
    Can enter subfolders.


Engine.CreateAppDataDir( path) Attempts to create a folder with the path path relative to your Mod's AppData folder.
Using ../ to go up a folder is NOT allowed.

Returns true if the operation was successful, false otherwise.


Engine.MoveAppDataDir( path, newPath) Attempts to move a folder with the path path relative to your Mod's AppData folder to a new location newPath also relative to your mod's AppData folder.
Using ../ to go up a folder is NOT allowed.

Returns true if the operation was successful, false otherwise.


Engine.RemoveAppDataDir( path, force = false) Attempts to delete a folder with the path path relative to your Mod's AppData folder.
Using ../ to go up a folder is NOT allowed.

  • force: If true, this function will forcefully remove a folder. Otherwise, a folder will only be removed if it is empty.
Returns true if the operation was successful, false otherwise.
